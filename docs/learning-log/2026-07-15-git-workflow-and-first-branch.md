# 학습 일지: Git 작업 흐름과 첫 작업 브랜치

날짜: 2026-07-15

단계: Phase 0 - 개발 작업 흐름 준비

진행 상태: 완료

## 오늘의 목표

`main`을 안정적인 통합 브랜치로 유지하고, 남은 Phase 0 조사를 별도의 작업 브랜치에서 진행한다.

## 왜 이 작업을 했는가

기능 개발과 검증된 결과를 분리하면 진행 중인 변경이 안정적인 기준 상태를 망가뜨리지 않는다. 혼자 개발하더라도 브랜치, 자체 리뷰와 Pull Request를 사용해 팀 개발의 변경 관리 절차를 연습한다.

## 진행한 활동

- 저장소가 `main` 브랜치에 있고 작업 트리가 깨끗한지 확인했다.
- `main`의 최신 커밋에서 `chore/phase-0-wsl-audit` 브랜치를 생성했다.
- 현재 브랜치가 새 작업 브랜치인지 확인했다.
- 브랜치 생성 후에도 작업 트리가 깨끗한지 다시 확인했다.
- 프로젝트에서 사용할 브랜치 이름, 커밋과 Pull Request 규칙을 문서화했다.

## 실행한 명령

```powershell
cd C:\project\turtlebot-fleet-ops
git status
git switch -c chore/phase-0-wsl-audit
git branch --show-current
git status
```

## 실제 결과

```text
시작 브랜치: main
시작 작업 트리: clean
생성한 브랜치: chore/phase-0-wsl-audit
현재 브랜치: chore/phase-0-wsl-audit
생성 후 작업 트리: clean
```

브랜치를 만든 직후에는 새 커밋이 없으므로 `main`과 작업 브랜치가 같은 커밋을 가리킨다.

## 배운 점 / 메모

- `main`은 프로젝트의 최종 완성본만 보관하는 곳이 아니라 현재 검증된 통합 상태를 보관하는 곳이다.
- 브랜치는 파일 전체를 복사하는 대신 특정 커밋을 가리키는 이름이다.
- 작업 브랜치를 만들면 진행 중인 변경을 `main`과 격리할 수 있다.
- 현재 규모에서는 장기 `develop` 브랜치보다 짧게 사용하는 작업 브랜치가 단순하고 목적이 명확하다.
- 브랜치 생성 자체는 파일 내용을 바꾸지 않으므로 별도 커밋이 생기지 않는다.

## 발생한 문제와 해결

문제 없이 브랜치를 생성했다. 작업 전후에 `git status`를 확인해 기존 변경사항이 섞이지 않았음을 확인했다.

## 복습 문제와 정답

### 1. `main`은 프로젝트 최종 완성본만 보관하는 브랜치인가?

정답: 아니다. 현재 시점에서 검증되고 다른 작업의 기준으로 사용할 수 있는 통합 상태를 보관한다.

이유: 작은 기능도 검증이 끝나면 `main`에 순차적으로 통합해야 항상 실행 가능한 기준점을 유지할 수 있다.

### 2. 작업 브랜치를 `main`에서 만드는 이유는 무엇인가?

정답: 검증된 최신 상태를 출발점으로 삼고, 진행 중인 변경을 `main`과 격리하기 위해서다.

이유: 문제가 생겨도 `main`은 영향을 받지 않으며 변경 범위를 Pull Request에서 명확히 비교할 수 있다.

### 3. Draft Pull Request와 일반 Pull Request의 차이는 무엇인가?

정답: Draft PR은 아직 작업 중임을 나타내며 병합할 수 없다. Ready for review로 전환한 PR은 정식 검토와 병합 대상이 된다.

이유: 작업 중인 변경을 일찍 공유하면서도 미완성 상태가 실수로 병합되는 것을 방지할 수 있다.

### 4. 현재 프로젝트에서 장기 `develop` 브랜치를 사용하지 않는 이유는 무엇인가?

정답: 현재는 한 명이 작은 기능을 순차적으로 개발하므로 장기 통합 브랜치가 추가 복잡성만 만들 가능성이 크기 때문이다.

이유: 짧은 작업 브랜치와 Pull Request만으로 변경 격리, 검토와 안정적인 `main` 유지가 가능하다.

### 5. 작업 브랜치는 언제 삭제하는가?

정답: Pull Request가 `main`에 병합되고 해당 브랜치에서 추가 작업할 필요가 없을 때 삭제한다.

이유: 병합된 기록은 커밋과 Pull Request에 남으므로 사용이 끝난 브랜치를 계속 유지할 필요가 없다.
