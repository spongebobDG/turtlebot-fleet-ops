# 학습 일지: 한 화면 관제와 heartbeat 단절 감사 로그

날짜: 2026-07-19

단계: Phase 4·5·6 운영 UX 보완

진행 상태: 구현 및 실환경 검증 완료

## 오늘의 목표

- 1280×720 관제 화면에서 페이지를 내리지 않고 TB1 상태, 지도, 제어 버튼과 최근 로그를
  동시에 확인한다.
- 갑작스러운 TB1 전원·통신·Agent 단절을 감사 이벤트로 남기고 복구 전이를 확인한다.
- 브라우저 종료와 Gateway lease 종료가 서로 다른 사건임을 운영 문서에 명확히 남긴다.

## 왜 이 작업을 했는가

기존 화면은 1280×720에서 문서 높이가 3,043px였다. 첫 화면에는 로봇 카드 일부만 보여
지도 목적지 제어와 감사 로그를 확인하려면 긴 페이지 스크롤이 필요했다. 또한
`online=false`는 화면에 표시됐지만 heartbeat 침묵과 복구가 Audit Events에 별도 전이로
남지 않아 사후 확인이 불편했다.

## 진행한 활동

1. 대시보드를 상단 요약과 3열 관제석으로 재배치했다.
2. 왼쪽에는 TB1 상태와 비상정지, 가운데에는 지도·초기 위치·목적지 제어, 오른쪽에는
   작업·활성 고장·감사 이벤트를 배치했다.
3. 페이지 자체는 viewport 높이에 고정하고 긴 기록만 각 목록 내부에서 스크롤하게 했다.
4. Gateway가 UI 접속 여부와 무관하게 registry snapshot을 주기적으로 확인하게 했다.
5. online 기준선 이후 단절에는 `ROBOT_OFFLINE`, 복구에는 `ROBOT_ONLINE`을 한 번씩
   SQLite 감사 로그에 저장하게 했다.
6. 관제 Zenoh 경로만 잠시 중단했다가 복구해 로봇을 움직이지 않고 실환경 전이를 검증했다.

## 실제 결과

- 브라우저 viewport: 1280×720
- 변경 전 문서 높이: 3,043px
- 변경 후 문서·body 높이: 720px
- 세 관제 패널 하단: 708px
- 연결 표시: `실시간 연결`
- TB1, 지도, Navigation 상태, 작업 8개, Fault 0개, Audit Events 50개 동시 표시
- 관제 통신 중단 뒤 `ROBOT_OFFLINE` ERROR 이벤트 확인
- 통신 복구 뒤 `ROBOT_ONLINE` INFO 이벤트 확인
- 반복 snapshot으로 같은 상태 이벤트가 증폭되지 않는 단위 테스트 추가
- Gateway 전체 pytest: 48 passed, 기존 pytest plugin 경고 2건, 실패 0
- 지도 JavaScript 테스트: 3 passed, 실패 0

실제 화면 메시지는 다음과 같다.

```text
ROBOT_OFFLINE  로봇 heartbeat가 끊겼습니다 (전원·네트워크·Agent 중단 가능)
ROBOT_ONLINE   로봇 heartbeat가 복구되었습니다
```

## 배운 점 / 메모

- 한 화면 관제는 모든 정보를 작게 만드는 것이 아니라 행동 빈도와 위험도에 따라 열을
  나누고 오래된 기록만 내부 스크롤로 격리하는 문제다.
- heartbeat 침묵은 현상이지 원인이 아니다. 전원, 네트워크, Agent 장애를 한 메시지로
  단정하면 잘못된 장애 분석이 된다.
- 브라우저 WebSocket과 NavigationLease의 소유자는 다르다. 브라우저 탭을 닫아도
  Gateway가 살아 있으면 목표는 계속되므로, 감독 종료는 명시적 취소나 e-stop이어야 한다.
- 관제 PC가 갑자기 꺼지면 꺼진 Gateway는 마지막 중앙 로그를 쓸 수 없다. 로봇 로컬의
  lease 만료와 재기동 뒤 durable task 실패 처리가 이 공백을 보완한다.

## 복습 문제와 정답

### 1. 브라우저 종료를 곧바로 로봇 정지 신호로 사용하지 않는 현재 이유는 무엇인가?

정답: 브라우저는 표시와 명령 입력 클라이언트이고, 실제 원격 소유권 lease는 Gateway가
발행하기 때문이다. 탭 새로고침이나 일시적인 UI 장애를 NavigationLease 장애로 해석하지
않는다.

### 2. offline event를 Robot Agent가 직접 보내지 않는 이유는 무엇인가?

정답: 전원이 꺼지거나 통신이 끊긴 Agent는 메시지를 보낼 수 없으므로 수신자인 Gateway가
마지막 heartbeat 이후 monotonic 경과 시간으로 판정해야 하기 때문이다.

### 3. 페이지 스크롤을 없애도 모든 감사 이벤트를 확인할 수 있는 이유는 무엇인가?

정답: 페이지 전체 높이는 고정했지만 Audit Events 카드의 목록은 독립적인 overflow 영역으로
남겨 최신 이벤트를 한눈에 보고 필요할 때 그 목록만 스크롤할 수 있기 때문이다.
