# 학습 일지: TB1 단일 로봇 MVP로 집중

날짜: 2026-07-16
단계: 프로젝트 범위 관리
진행 상태: 범위 결정과 문서 반영 완료

## 결정

현재 완료 목표를 TB1 한 대의 운영 가능한 웹 관제·안전 주행·SLAM·Nav2·작업·장애 관리
MVP로 좁혔다. TB2 실차와 다중 로봇 할당은 TB1 완료 뒤의 backlog로 이동했다.

## 오늘 한 일

- README의 목표·단계·아키텍처·다음 작업을 현재 범위에 맞췄다.
- 권위 있는 범위와 acceptance 조건을 `docs/design/project-scope.md`에 작성했다.
- 기존 TB2 준비 결과는 삭제하지 않고 향후 확장 증거로 보존했다.
- 방금 성공한 TB1 10cm 매핑 구간과 pose graph·known cell 증가를 학습 일지에 기록했다.

## 오늘 꼭 기억해야 할 것

1. **범위 축소는 실패가 아니라 검증 가능한 수직 기능을 먼저 완성하는 우선순위 결정이다.**
2. **현재 완료 범위와 향후 확장을 문서에서 명시적으로 분리한다.**
3. **실행하지 않은 TB2 기능을 완료라고 주장하지 않되 재사용 가능한 계약과 증거는 보존한다.**
4. **다음 세션은 마지막 안전 상태와 남은 acceptance 조건에서 이어가야 한다.**

## 면접에서는 이렇게 설명한다

> 두 대를 동시에 부분 구현하기보다 TB1 한 대의 전체 운영 경로를 먼저 완성하기로 범위를
> 조정했습니다. 안전, 상태, 통신, 지도, 자율주행, 작업과 장애 기록을 실차와 CI로 검증하고,
> TB2는 검증된 계약을 namespace와 robot ID로 격리해 복제하는 후속 단계로 남겼습니다.

## 복습 문제와 정답

### 1. 범위를 줄일 때 반드시 함께 바꿔야 하는 것은 무엇인가?

정답: 목표, in/out scope, 완료 조건, roadmap, 문서의 완료 주장과 backlog다.

### 2. 다음 세션의 첫 작업은 무엇인가?

정답: 새 물리 이동부터 하지 않는다. TB1 서비스·e-stop·0속도·입력 Publisher·SLAM 상태를
재확인한 뒤 loop closure를 위한 짧은 구간을 새 현장 승인과 함께 진행한다.

## 다음 세션 시작점

1. TB1 서비스와 최종 안전 상태 재검증
2. 출발 지점 재관측 경로를 짧은 보호 이동으로 수행해 loop closure 확인
3. 지도 YAML·PGM·pose graph 저장, round-trip과 시각 검수
4. AMCL과 터미널 Nav2 Goal 실차 acceptance 시작
